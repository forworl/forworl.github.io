HashMap详解

发表于2023-12-10|更新于2023-12-10|Java SE
|字数总计:5k|阅读时长:22分钟|阅读量:

HashMap

jdk1.7实现

数组 + 链表

Entry结构
初始化

初始数组大小默认为16个，负载因子为3/4，数组达到负载因子则进行扩容

table[]数组等待需要时再创建

put()
  1. 如果table[]为空，创建table[]数组
  2. 如果key == null，会在下标为0位置放入
  3. hash出 key 的对应下标
  4. 如果下标位置没有元素，直接插入
  5. 否则判断首元素是否是当前元素，是的话，进行 value 替换
  6. 否则，进行链表插入，如果节点个数到达阈值，先扩容，再头插法添加元素
get()
  1. 如果key == null，会在下标为0位置查询
  2. hash计算出 key 对应下标
  3. 如果下标位置首元素符合，返回结果
  4. 否则判断后一个元素是否存在，不存在直接返回null
  5. 否则遍历链表
  6. 找不到返回null
扩容resize()
死链问题

扩容的时候线程并发产生的循环引用，主要是前插法同时线程并发导致的

数据覆盖问题

jdk1.8实现

数组 + 链表 + 红黑树

Node结构
初始化

初始数组大小默认为16个，负载因子为3/4，数组达到负载因子则进行扩容

第一次调用put()，调用resize()方法创建table[]数组

put()
  1. 如果table[]为空，创建table[]数组
  2. hash出 key 的对应下标
  3. 如果下标位置没有元素，直接插入
  4. 否则判断首元素是否是当前元素，是的话，进行 value 替换
  5. 否则判断首元素是否是TreeNode，是的话，红黑树插入
  6. 否则，进行链表插入，添加完之后判断是否需要树化
  7. ++size > threshold，判断是否需要扩容
红黑树插入putTreeVal()
get()
  1. hash计算出 key 对应下标
  2. 如果下标位置首元素符合，返回结果
  3. 否则判断后一个元素是否存在，不存在直接返回null
  4. 后一个元素是否是TreeNode，是的话，红黑树搜索
  5. 否则遍历链表
  6. 找不到返回null
红黑树搜索getTreeNode()
扩容resize()

如果是初始化，会默认初始化容量和加载因子

正常扩容的话，遍历Entry数组

  1. 当前下标的首元素后面没有链表或者树，直接插入
  2. 首元素后面是树的话，执行split()
  3. 首元素后面是链表的话，根据e.hash & oldCap) == 0，将链表分成lo，hi两个链表，分别放在下标为 j 和 j + oldCap位置
树节点的结构TreeNode
树化treeifyBin()
  1. 遍历原链表，用每个节点创建一个TreeNode，来创建一个TreeNode链表
  2. 遍历TreeNode链表，把每个TreeNode加入到红黑树当中
  3. 没有根节点，直接创建新的根节点，颜色为黑色
  4. 否则从根节点开始遍历，对比当前节点和叶子节点的hash值得出不同的dir，判断叶子节点插入当前节点左边（-1）还是右边（1），并且要进行左旋或者右旋，来维护红黑树的特性
  5. 确保给定的根节点是所在桶的第一个节点
维护红黑树特性balanceInsertion()

以下红黑树部分参考HashMap源码分析putTreeVal（红黑树部分）——java学习总结（11）-CSDN博客

无需调整 【变色】即可实现平衡 【旋转+变色】才可实现平衡
(1)当父节点为黑色时直接插入子节点 (2)空树插入根节点，将根节点红色变为黑色 (4)父节点为红色左节点，叔父节点为黑色，插入左子节点，那么通过【左左节点旋转】
(3)父节点和叔父节点都为红色，则将父节点和叔父结点置为黑色，再将祖父结点置为红色，再递归向上处理（将祖父结点视为新插入的结点，将父节点和叔父节点看作黑色的null） (5)父节点为红色左节点，叔父节点为黑色，插入右子节点，那么通过【左右节点旋转】
(6)父节点为红色右节点，叔父节点为黑色，插入左子节点，那么通过【右左节点旋转】
(7)父节点为红色右节点，叔父节点为黑色，插入右子节点，那么通过【右右节点旋转】

(4)左左节点旋转

(5)左右节点旋转

(6)右左节点旋转

(7)右右节点旋转

数据覆盖问题

同1.7

JDK1.7 和 JDK1.8 的区别

底层数据结构不同，1.7使用的是Entry数组 + 链表，1.8使用的是Node数组 + 链表 + 红黑树

链表插入方式不同，1.7使用头插法（有循环死链问题），1.8使用尾插法，都有数据覆盖的问题

处理hash的时候，hash扰乱的次数不同（hash函数里面的异或），1.7有4次，1.8只有2次

对key == null 的处理有些许差别，1.7会单独处理key == null，1.8和在hash函数里面处理，不单独处理
