JVM虚拟机

发表于2023-10-22|更新于2023-10-23
|阅读量:

内存结构

  • 线程私有：程序计数器、虚拟机栈、本地方法区
  • 线程共享：堆、方法区, 堆外内存（Java7的永久代或JDK8的元空间、代码缓存）

线程私有：它的生命周期随着线程的创建而创建，随着线程的结束而死亡

程序计数器：记住下一条指令的地址（偏移地址）

  • 线程私有
  • 执行本地方法，值为空
  • 不会内存溢出OutOfMemoryError

虚拟机栈：java程序控制台看到的堆栈信息

操作数栈 用于存放方法执行过程中产生的中间计算结果。另外，计算过程中产生的临时变量也会放在操作数栈中。

动态链接 主要服务一个方法需要调用其他方法的场景。Class 文件的常量池里保存有大量的符号引用比如方法引用的符号引用。当一个方法要调用其他方法，需要将常量池中指向方法的符号引用转化为其在内存地址中的直接引用。动态链接的作用就是为了将符号引用转换为调用方法的直接引用，这个过程也被称为 动态连接 。

栈内存溢出可能原因：

  • CPU占用过高，程序一直在运行

  • 死锁

  • 特点？

  1. 栈是一种快速有效的分配存储方式，访问速度仅次于程序计数器
  2. JVM 直接对虚拟机栈的操作只有两个：每个方法执行，伴随着入栈（进栈/压栈），方法执行结束出栈
  3. 栈不存在垃圾回收问题
  4. 可以通过参数-Xss来设置线程的最大栈空间，栈的大小直接决定了函数调用的最大可达深度
  • 该区域有哪些异常？
  1. 如果采用固定大小的 Java 虚拟机栈，那每个线程的 Java 虚拟机栈容量可以在线程创建的时候独立选定。如果线程请求分配的栈容量超过 Java 虚拟机栈允许的最大容量，Java 虚拟机将会抛出一个 StackOverflowError 异常
  2. 如果 Java 虚拟机栈可以动态扩展，并且在尝试扩展的时候无法申请到足够的内存；或者在创建新的线程时没有足够的内存去创建对应的虚拟机栈，那 Java 虚拟机将会抛出一个OutOfMemoryError异常

本地方法栈：用c写的程序去执行一些本地方法

虚拟机栈为虚拟机执行 Java 方法 （也就是字节码）服务，而本地方法栈则为虚拟机使用到的 Native 方法服务

特点和虚拟机栈类似，也有栈帧，也会抛异常，栈深度溢出StackOverflowError 和栈扩展失败OutOfMemoryError

如何访问对象

  • 句柄访问：reference指向包含对象实例数据、类型数据的指针，需要二次访问真正的数据，但是GC之后不用修改reference本身
  • 直接访问：reference直接指向对象，速度快

OOM异常原因

堆溢出：

  • 内存泄露
  • 内存溢出：某些对象生命周期过长

栈溢出：

  • StackOverflowError：栈容量固定大小，栈帧太大或者栈容量太小
  • OutOfMemoryError：栈容量动态拓展，不断创建线程，每个线程分配的栈内存大

方法区和运行时常量池溢出：

CGLib字节码技术增强和动态语言

本机直接内存溢出：

默认大小和堆大小一样，直接用了Unsafe类分配内存，信息较少

垃圾回收GC

哪些对象可以被回收？

根对象（核心系统类、本地对象、线程、加锁的对象）不能被回收

  • 软弱引用：在没有强引用时会被垃圾回收，放入引用队列
  • 虚引用：第一次先入队，释放ByteBuffer，入队后，另一个线程查询直接内存地址，调用unsafe释放直接内存
  • 终结器引用：第一次先入对，入队后，另一个线程查询是否有终结器引用，调用finallize，下一次垃圾回收释放内存

引用本身也要占用内存，当内存不够时，垃圾回收可能回收掉弱引用引用的资源和弱引用本身

垃圾回收次数和堆的大小有关

伊甸园满了，新生代垃圾回收，老年代垃圾回收，如果两次垃圾回收都无法提供出足够大的内存，就会OOM，单个线程的OOM不h会让程序暂停

垃圾回收器

一次STW可以由多次垃圾回收组成

垃圾回收方式

  1. 串行：单线程、堆内存小：只由一个垃圾回收线程进行垃圾回收
  2. 吞吐量优先：多线程、单位时间内，STW时间最短：多个垃圾回收线程同时垃圾回收，CPU占用率高
  3. 响应时间优先：多线程、单次STW时间最短：只由一个垃圾回收线程进行并发垃圾回收，但是标记是所有线程共同执行，初始标记会第一次阻塞（STW），然后重新标记第二次阻塞（STW）（因为并发标记的时候可能对象发生改变）

内存模型

原子性：读写之间有间隔时间，其他线程会修改值 （i++）

可见性：一个线程对变量的修改对另一个变量是可见的，缓存和内存之间的数据不一致问题，解决办法： 变量加上volatile关键字保证可见性，强制读内存
